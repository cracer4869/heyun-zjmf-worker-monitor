# 核云监控审计问题修复设计

**项目名称：** heyun-zjmf-worker-monitor
**原项目目录：** D:\\05.天才程序员\\16.自建网站\\01.复刻网站\\01号
**内容类型：** 计划
**日期：** 2026-07-13
**关键词：** EdgeOne、Cloudflare、GitHub Actions、重启限额、状态页

## 目标

让 GitHub `main` 能够独立重建 EdgeOne 版本，消除失败重启无限重试，避免 HTTP 业务故障被电源 API 状态掩盖，收敛重复调度和公开状态数据，并修正 Cloudflare/GitHub 发布配置。

## 设计

1. 从上游同项目补齐 EdgeOne 运行时依赖，保持现有模块边界和 API。
2. 将重启尝试计数纳入统一限额查询；成功计数继续保留用于展示，失败尝试也必须阻止后续动作。
3. `http_then_api` 与三步探测仅在 HTTP/TCP 服务探测成功时报告健康；电源 API 只决定开机/重启动作，不把主机通电等同于业务可用。
4. 只保留主仓库的 Bootstrap/Loop 调度，删除私有仓库重复 Cron；租约增加基于 owner 的释放校验，并在测试中覆盖并发入口。
5. 匿名状态接口只返回脱敏状态、汇总历史和有限事件字段；管理接口继续使用 Bearer Token。
6. Cloudflare 使用可解析的真实 D1 绑定配置，部署工作流使用最小权限；不在本次变更中删除生产数据或轮换密钥。

## 验收

- EdgeOne 和 Cloudflare 测试全部通过。
- 新增失败重启限额、HTTP 504 误报、状态脱敏和租约行为测试。
- `wrangler.toml` 不再含占位 D1 ID；Actions 工作流静态校验通过。
- 未授权管理接口仍返回 401，匿名状态接口继续返回 200。
